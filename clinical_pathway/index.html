Clinical Pathway

Clinical pathway is a structured data entry where questions related to a particular problem will be grouped and organized to facilitate input.

Problem List

This is the entry point to the application where the user select a problem which closely matches the user complain. Each case can have a single or multiple problem. For discussion purpose I will choose ‘Sore Throat’ as an example. Below is an example of a problem list.

Main Input

The main input page consist of a sidebar for navigation on the left and the clinical pathway on the other side. Here the inputs are broken down based on the selected problem attributes. For an example, sore throat can be defined by onset, duration or nature. The attributes will be grouped into different headers such as History of present illness, Follow-up or Exacerbating factor. Finally they will be placed into one of the primary options of Subjective, Objective or Assessment & Plan (SOAP).
